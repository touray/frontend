3.4.2 #frontend.mixins.grid CSS Grid

Toggle full screen Open in new window Toggle example guides Toggle HTML markup

Creates a CSS grid container. View in Firefox inspector to see a grid overlay.

Example
  • $grid-columns: responsive list
    Default: $css-grid-columns
    Defines the number of grid columns.
  • $grid-rows: responsive list
    Default: (small: none, medium: none)
    Defines the number of grid rows.
  • $grid-column-gaps: responsive list
    Default: $css-grid-column-gaps
    Specifies the size of the grid lines between the columns.
  • $grid-row-gaps: responsive list
    Default: $css-grid-row-gaps
    Specifies the size of the grid lines between the rows.
  • $type: grid|inline-grid|subgrid
    Default: grid
    Defines the grid formatting context.
@include css-grid($grid-columns, $grid-rows, $grid-column-gaps, $grid-row-gaps, $type);

3.5 #frontend.variables Variables

The variables file for Frontend Build components. Foundation is used by default in Frontend Build, so uses it's settings to define many of the default Frontend Build component variables. It's best to edit those variables from Foundation's scss/framework/foundation/_settings.scss. All other variables and custom variables should be defined in the scss/_variables.scss file.

Globals

  • $global-width: $global-width (Foundation setting)
  • $global-margin: $global-margin (Foundation setting)
  • $global-padding: $global-padding (Foundation setting)
  • $global-radius: $global-radius (Foundation setting)
  • $global-weight-bold: $global-weight-bold (Foundation setting)

3.6.1 #frontend.wordpress.wysiwyg WYSIWYG Editor

When you are working with WordPress WYSIWYG Editor in the Edit Post screen, WordPress applies context-specific CSS attributes to certain elements when they are included in your post. For example, when inserting images, you can specify whether the images are floated left, right, or even centered. To do this, WordPress applies classes such as alignleft, alignright, or aligncenter. Without defining styles for WordPress default hooks, images and other elements may not display as intended.

Thus, at a bare minimum, if you are building a theme that will be used by the general public, it is wise to include the following basic styles for the items attributed via the WYSIWYG Editor.
